دليل المبرمج لتطوير النظام: من نظام إدارة طلاب إلى منصة متكاملة (ERP)
بناءً على شرحك للهيكل الحالي لنظامك، سأقدم لك خريطة طريق تفصيلية لتوسيعه إلى المنصة المتكاملة التي وصفناها سابقاً. سأركز على الجوانب المعمارية، توسيع قاعدة البيانات، إعادة هيكلة الأكواد، إضافة الوحدات الجديدة، آليات الأمان والصلاحيات، والتكامل مع الخدمات الخارجية. الهدف هو أن يكون لديك دليل عملي يرشدك في كتابة الأكواد وتنظيم المشروع.

4. نظام الصلاحيات المتقدم (RBAC) وتطبيقه
4.1. Decorator للتحقق من الصلاحيات
بدلاً من @login_required البسيط، سننشئ @permission_required يتحقق من أن المستخدم لديه صلاحية معينة (أو أحد الأدوار التي تملكها).

python
# app/utils/decorators.py
from functools import wraps
from flask import flash, redirect, url_for
from flask_login import current_user

def permission_required(permission_name):
    def decorator(f):
        @wraps(f)
        def decorated_function(*args, **kwargs):
            if not current_user.is_authenticated:
                flash('Please log in.', 'warning')
                return redirect(url_for('auth.login'))
            # تحقق من وجود الصلاحية في أدوار المستخدم
            user_permissions = set()
            for role in current_user.roles:
                for perm in role.permissions:
                    user_permissions.add(perm.name)
            if permission_name not in user_permissions:
                flash('You do not have permission to access this page.', 'danger')
                return redirect(url_for('main.index'))
            return f(*args, **kwargs)
        return decorated_function
    return decorator

نفصل المنطق المعقد في طبقة خدمات. مثلاً، خدمة محاسبية تقوم بإنشاء قيد تلقائي عند إصدار فاتورة.

python
# app/services/financial_service.py
from app.models.finance import JournalEntry, JournalLine, Account

def create_invoice_journal_entry(invoice):
    """إنشاء قيد محاسبي من فاتورة مبيعات"""
    # حساب الإيراد
    revenue_account = Account.query.filter_by(code='4000').first()  # حساب الإيرادات
    receivable_account = Account.query.filter_by(code='1100').first()  # حساب العملاء

    entry = JournalEntry(
        entry_number=f"INV-{invoice.invoice_number}",
        date=invoice.date,
        description=f"فاتورة مبيعات {invoice.invoice_number}",
        created_by=current_user.id
    )
    db.session.add(entry)
    db.session.flush()  # للحصول على ID

    # سطر مدين: حساب العميل (أو النقدية)
    line1 = JournalLine(
        entry_id=entry.id,
        account_id=receivable_account.id,
        debit=invoice.total,
        credit=0,
        description="قيمة الفاتورة"
    )
    # سطر دائن: حساب الإيرادات
    line2 = JournalLine(
        entry_id=entry.id,
        account_id=revenue_account.id,
        debit=0,
        credit=invoice.total,
        description="إيرادات مبيعات"
    )
    db.session.add_all([line1, line2])
    db.session.commit()
يتم استدعاء هذه الخدمة من داخل blueprint بعد إنشاء الفاتورة.

تجميع البيانات (Aggregation): بدلاً من استدعاء count() لكل علاقة، استخدم استعلامات تجميعية.

10. التقارير المتقدمة
10.1. منشئ التقارير المخصص
يمكنك بناء واجهة تسمح للمستخدم باختيار الحقول والفلاتر، ثم ديناميكياً بناء استعلام SQLAlchemy وتصدير النتيجة. استخدم مكتبة مثل Pandas لتحويل البيانات إلى DataFrame ثم إلى Excel/PDF.

10.2. تقارير PDF باستخدام قوالب HTML
بدلاً من Word، استخدم HTML/CSS ثم حولها إلى PDF باستخدام WeasyPrint أو pdfkit. هذا أسهل لتنسيق التقارير المعقدة.

مثال:

python
from weasyprint import HTML
html = render_template('reports/invoice_template.html', invoice=invoice)
pdf = HTML(string=html).write_pdf()
response = make_response(pdf)
response.headers['Content-Type'] = 'application/pdf'
return response
